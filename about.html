A small productivity workspace. A set of tools, one folder, one local server.

Everything lives on your laptop. Claude reads and writes the same JSON files the web UI does. The server just keeps them in sync and quietly backs them up every time something changes.

The tools

Each one solves a specific moment in the week. They share one folder and cross-pollinate so you're never re-typing the same action twice.

Daily driver

Plan & Track

The backbone. Your This-week list, Next steps, Timeline of weekly reports, meeting Sessions, and ongoing Initiatives, all in one page with subpages per area.

  • This week · active tasks, drag to reorder
  • Next steps · captured actions with owner + source
  • Sessions · meeting notes, one block per topic
  • Timeline · archive of weekly reports only
  • Initiatives · long-running work streams
Review sessions

Show & Note

Drop a prototype, demo it live, capture feedback + decisions + next steps in one pass. Next steps go back into Plan & Track automatically.

  • Renders .jsx, .html inline
  • Capture panel · feedback · decisions · next steps
  • Export any session as a shareable .md
  • Past sessions list with notes searchable
Post-vacation

Context Catch-up

Coming back from time away? Drop everything from the missed window into one folder, run the Cowork prompt, and get an importance-ordered triage list you can burn down.

  • Vacation dates set the analysis window
  • Claude reads the folder and writes the list
  • Each item has a source pill back to its origin
  • "Add next steps" promotes items into Plan & Track

How an item moves through the system

An action item doesn't belong to one tool. It has a lifecycle. Here's the path it takes from "someone mentioned a thing" to "it's done, archived for the weekly report."

1 · Capture · wherever it happens
From a meeting Transcript lands in files/transcripts/. The 4:00pm automation extracts action items into Sessions + Next steps.
From a demo You type it into a Show & Note session's Next steps box. It's saved against that session.
From a vacation window Claude extracts it from the catch-up folder. You click "Add next steps" on the item.
↓
2 · Next steps · the holding pen
Plan & Track → Next steps Every capture lands here first, tagged with its source (Session name, Show & Note session, Catch-up: file). Nothing has to move yet.
↓
3 · Promote · when you're ready to do it
This week You drag (or click) to promote a Next step into This week. Detail + source come along.
↓
4 · Archive · the weekly rollover
Check it off Done items stay on their own list (struck through) until the Friday rollover archives them.
Friday 5:00pm Weekly wrap-up runs: a report is generated, saved, and This week is rolled over so Monday starts clean.
Timeline One entry per week with the full report markdown. The Timeline only holds weekly reports, not individual items.

The automations

Cron jobs that run the repetitive stuff. Each one is just a prompt Claude executes against your data. You can copy the prompt from the UI and run it ad-hoc in any Cowork chat.

Where to find the prompt · open the relevant tool page and expand the Automations card at the top. Each automation exposes its full Cowork prompt and setup guidance, ready to copy into a fresh chat or hand to Claude.

Process new transcripts

Looks in files/transcripts/ for .vtt or .txt files that don't yet have a matching Session, extracts topic blocks + decisions + action items, and appends them to Plan & Track. Safe to re-run.

Prompt lives on the Plan & Track · Sessions page.

Mon–Fri · 4:00pm

Import this week's meetings

Reads the latest calendar screenshot in files/calendar/ and creates a Session per meeting for the week. Skips anything already present. Needs a fresh screenshot dropped in, manually or via an OS automation.

Prompt lives on the Plan & Track · Sessions page.

Mondays · 9:00am

Weekly wrap-up

Generates the weekly report in canonical format, displays it, saves it to files/weekly-summaries/ and the Timeline, then rolls done This-week items off so Monday starts empty.

Prompt lives on the Plan & Track · Timeline page.

Fridays · 5:00pm

Post-vacation catch-up

Reads everything in files/Context-catch-up/, extracts what matters across your vacation window, and writes an importance-ordered list to the Catch-up tool. On-demand only, not scheduled.

Prompt lives on the Context Catch-up page.

Data & safety

Three JSON files are the source of truth. Everything else is derived, input material, or a safety net.

data/plan-and-track.json
Source of truth Everything Plan & Track shows: This week, Next steps, Timeline, Sessions, Initiatives, weekly reports, your name/role, theme.
data/sessions.json
Source of truth Show & Note history: every past session with its feedback, decisions, and next steps.
data/context-catchup.json
Source of truth The current catch-up: vacation period + importance-ordered items.
data/backups/
Safety net Last 20 rolling saves per JSON file. Auto-written every time you save.
data/vault/
Safety net One permanent snapshot per file per day. Never deleted.
files/transcripts/
Input Drop your .vtt / .txt meeting transcripts here.
files/Context-catch-up/
Input Drop everything from your time-away window before running the Catch-up prompt.
files/prototypes-and-presentations/
Input Things you want to open in Show & Note: .jsx, .html, .md, .svg, images.
files/weekly-summaries/
Derived Convenience .md copies of weekly reports. Canonical content is inside plan-and-track.json.
files/calendar/
Input Weekly calendar screenshot for the Monday import automation.
Rule of thumb · if you delete the three source-of-truth JSON files, the tools are empty. If you delete anything else, they still work. You've just lost input material or safety copies.
